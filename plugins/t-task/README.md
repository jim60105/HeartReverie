# t-task

質感任務（Texture Task）寫作指令系統，定義極致感官細節特寫的寫作規範與思考框架。

## 運作原理

外掛注入兩段提示詞片段：

| 片段 | Priority | 說明 |
|------|----------|------|
| `t_task` | 100 | 完整 T-task 規格，定義六種場景類型、十條寫作規則、元素表、詞彙系統 |
| `t_task_think_format` | 100 | 思考分析模板，用於場景解析與寫作節點規劃 |

LLM 輸出的質感段落以 `<T-task>` 標籤包裹，與一般敘事 `<ordinary>` 區分。外掛透過正規表達式從提示詞與前端顯示中移除帶有屬性的 `<T-task...>` 變體標籤（如 `<T-task情景分析>`），同時保留裸標籤 `<T-task>` 內的實際故事內容。

### 寫作規則摘要

1. 實際感官描寫，禁用「好像」、「彷彿」等虛化詞
2. 純物理描寫，不加入美醜判斷
3. 長短句交替控制節奏
4. 多重修飾詞疊加
5. 至少涵蓋兩種感官
6. 直接解剖學用語，禁用委婉表達
7. 多元素覆蓋（顏色、形狀、溫度、濕度、氣味、聲音、質感）
8. 禁止句式重複
9. 「鉤子細節」原則——一個只有近距離才能注意到的超寫實細節
10. 創意反重複——避免重複先前使用過的角度與手法
