# status

從 LLM 輸出中提取角色狀態面板，解析結構化格式後在前端渲染為帶圖示的可摺疊角色資訊卡。

## 運作原理

外掛透過 `promptFragments` 注入提示詞指令，要求 LLM 在回覆中以 `<status>` 標籤輸出女性角色的狀態面板，包含三個區段：基礎資訊、服飾、特寫。

前端模組提取 `<status>...</status>` 區塊，解析各區段欄位，轉換為帶 emoji 圖示的資訊卡面板。

### 狀態格式

LLM 輸出範例：

```
<status>
[Basic|小夜|圖書館助理|舊書庫一角|這封信到底是誰留下的……|手電筒、舊鑰匙]
[Outfit|白襯衫搭深藍長裙|黑色平底鞋|膚色絲襪|細框眼鏡、髮圈]
[Close-up|指尖|沾著舊書頁的灰塵，微微發抖|嘴唇|因緊張而輕咬下唇|眼神|透過鏡片專注盯著信上的字跡]
</status>
```

### 渲染元件

| 區段 | 欄位 | 圖示 |
|------|------|------|
| 基礎 | 場景 | 📍 |
| 基礎 | 心聲 | 💭 |
| 基礎 | 持有物 | 👜 |
| 服飾 | 衣著 | 👚 |
| 服飾 | 鞋子 | 🧦 |
| 服飾 | 襪子 | 💍 |
| 服飾 | 配飾 | ⛓️ |
| 特寫 | 細節 | 🔍 |

服飾與特寫區段為可摺疊設計。解析失敗時回退為 `<pre>` 原文顯示。
