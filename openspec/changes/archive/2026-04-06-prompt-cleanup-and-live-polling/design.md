# Design — Prompt Cleanup & Live Polling

## D1: Tag stripping with regex

Strip `<options>...</options>` and `<disclaimer>...</disclaimer>` (including all enclosed content, multiline) from chapter text before wrapping in `<previous_context>`.

- Use a single regex with the dotAll (`s`) flag so `.` matches newlines: `/<options>[\s\S]*?<\/options>|<disclaimer>[\s\S]*?<\/disclaimer>/g`
- Apply per-chapter (inside `chapters.map()`), not globally across the joined string.
- Implement as a named helper `stripPromptTags(content)` for readability and testability.
- Trim the result after stripping to remove leading/trailing whitespace left behind by removed tags.

Rationale: `<options>` and `<disclaimer>` are UI artifacts generated by the model. They waste tokens and can confuse subsequent prompt context. Stripping them keeps the prompt focused on actual story content.

## D2: Content-aware polling

Current `pollBackend()` only checks whether the chapter *count* has changed. This misses in-progress streaming updates to the last chapter file.

New behaviour:
- On each poll tick, also fetch the last chapter's content via `GET /api/stories/:series/:name/chapters/:number`.
- Compare the fetched content with `state.backendChapters[lastIndex].content`.
- If different, update `state.backendChapters[lastIndex].content` in place.
- If the user is currently viewing that chapter (`state.currentIndex === lastIndex`), re-render the chapter display.

## D3: Efficient polling

- Only fetch a **single** chapter's content per poll tick (the last one), not all chapters.
- Continue checking chapter count as before; if count changed, perform a full reload via `loadFromBackend()`.
- Content comparison is a simple string equality check — no diff algorithm needed.
- The existing 1-second poll interval is retained.
