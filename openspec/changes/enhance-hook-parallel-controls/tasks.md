## 2. Code: registration-time throttle warning

- [x] 2.1 In `HookDispatcher.register()` (`writer/lib/hooks.ts`), after the new `HandlerEntry` has been pushed onto the per-stage list and the list re-sorted by priority, add a new private helper `#warnIfHeterogeneousConcurrency(stage, list, newEntry)` invoked only when `newEntry.parallel === true`.
- [x] 2.2 In the helper, partition the per-stage list into parallel-bucket entries (`e.parallel === true`). Skip the check entirely if there are fewer than 2 parallel entries (no throttle relationship yet).
- [x] 2.3 Within the helper, evaluate the new entry's `concurrency` declaration (call it `nc`, where `nc === undefined` means "no declaration / unbounded") against every other parallel entry on the stage (call each one's value `pc`). Detect a mismatch in either of two cases: **(a) finite-vs-unbounded** — exactly one of `(nc, pc)` is `undefined` and the other is a finite integer; **(b) finite-vs-higher-finite** — both `nc` and `pc` are finite integers AND `nc !== pc`. If no existing parallel entry produces a mismatch against the new entry, the bucket is homogeneous from the new entry's perspective — return without logging.
- [x] 2.4 Build the warning payload from the set of mismatching (new-entry, existing-entry) pairs collected in 2.3. Identify the "throttling" side(s) as the entries whose declared `concurrency` is the strict minimum among the involved entries (a `undefined`/unbounded peer is never the throttler when matched against a finite peer; between two finite peers the lower value is the throttler). Identify the "slowed" side(s) as the other entries in those pairs. The payload SHALL include the stage name, every throttling plugin's name and declared `concurrency` value, and every slowed plugin's name and either its declared `concurrency` value or the literal `"unbounded"` when omitted. Message text MUST include the literal phrase `throttle the entire '<stage>' parallel bucket`.
- [x] 2.5 Add a module-scoped `const throttleWarnDedup = new Set<string>()` near the existing `PARALLEL_ALLOWED` declaration. Before emitting the prepared warn, compute the key `${stage}::${newEntry.plugin ?? "<anonymous>"}::${newEntry.concurrency ?? "none"}`; if the set already contains it, return without logging. Otherwise insert it and emit. The set is intentionally never cleared (process-lifetime dedup) — document this in a `//` comment referencing the spec.
- [x] 2.6 Export a test-only reset helper (e.g. `_resetThrottleWarnDedupForTesting()`) so unit tests can clear the suppression set between cases. Mark it with a `/** @internal */` JSDoc and the existing naming convention used by other test-only exports in `writer/lib/`.
- [x] 2.7 Use `baseLogger?.warn(...)` when a plugin logger is attached on the new entry; otherwise fall back to the module-level `log.warn`. Match the log shape used by other dispatcher warnings (structured fields `plugin`, `stage`, `concurrency`, `unboundedPeers`, plus a human-readable `message`).
- [x] 2.8 Add `// Keep this check O(handlers per stage); no I/O.` comment per AGENTS.md guidance on hot-path checks.
